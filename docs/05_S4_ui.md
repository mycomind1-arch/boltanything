## S4.3 — `lib/anything/realtime.ts`

Single exported function `subscribeToRealtime(supabase, buildId, handlers)`. Returns an unsubscribe function.

```typescript
type RealtimeHandlers = {
  onBuildUpdate:   (row: BuildRow)    => void;
  onAgentUpdate:   (row: AgentRow)    => void;
  onJobUpdate:     (row: JobRow)      => void;
  onAiolMessage:   (row: AiolRow)     => void;
  onFileUpdate:    (row: FileRow)     => void;
  onVaultEntry:    (row: VaultRow)    => void;
};
```

Uses `supabase.channel('build:' + buildId)` with `postgres_changes` filters per table, all filtered by `build_id=eq.<buildId>`. For `aiol_messages` uses `id=gt.<lastSeenId>` window to avoid replaying old messages.

**One channel per active build.** On buildId change, unsubscribe old channel before subscribing new one.

## S4.4 — `lib/anything/use-build-stream.ts`

Primary state hook. Combines snapshot hydration + Realtime deltas.

```typescript
interface BuildStreamState {
  build:    BuildRow | null;
  agents:   AgentRow[];
  jobs:     JobRow[];
  messages: AiolRow[];
  files:    FileRow[];
  vault:    VaultRow[];
  status:   'idle' | 'loading' | 'live' | 'terminal';
}
```

**Lifecycle:**
1. `status='loading'` — calls `builds.snapshot({buildId})` to hydrate initial state
2. `status='live'` — opens Realtime channel; applies deltas to local state
3. `status='terminal'` — when `build.status ∈ {succeeded,failed}` AND no pending jobs remain; closes channel
4. On remount with same `buildId` — skip snapshot, open fresh Realtime channel (messages already in state from prior render)

**Delta rules:**
- Builds/agents/jobs/files: upsert by `id` into local array
- `aiol_messages`: append-only; deduplicate by `id`
- `vault_entries`: prepend to array

## S4.5 — `lib/anything/use-chat.ts`

```typescript
interface ChatState {
  messages: ChatMessage[];
  status: 'idle' | 'streaming' | 'tool_calling' | 'error';
  activeBuildId: string | null;
}

interface ChatMessage {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  toolCall?: { name: string; buildId: string };
  ts: string;
}
```

**On send:**
1. Append optimistic user message
2. `fetch('/api/chat', {method:'POST', body: JSON.stringify({messages})})` → SSE stream
3. Parse SSE chunks:
   - `data: {"type":"text","delta":"..."}` → append to last assistant message
   - `data: {"type":"build_started","buildId":"..."}` → set `activeBuildId`; `useBuildStream` activates
   - `data: {"type":"done"}` → status='idle'
   - `data: {"type":"error","message":"..."}` → status='error'

## S4.6 — `components/chat/ChatPane.tsx`

**Left pane. Three zones:**

**Header:** "Anything" wordmark + settings gear (→ admin page in S6).

**History sidebar** (collapsible on mobile): recent builds list. Each item shows directive (truncated 40 chars), status pill, relative time. Click → activate that build in `useBuildStream`.

**Composer:** `Textarea` (auto-resize, max 6 rows) + send button. Disabled while `status='streaming'`. `Cmd/Ctrl+Enter` to send. Placeholder rotates through 3 example directives on idle.
